# Technical Specification: CIE-10 Diagnosis System (ICD-10) ## 1. Database Schema (PostgreSQL) ```sql CREATE TABLE CIE10_Catalog ( codigo VARCHAR(10) PRIMARY KEY, -- Ej: 'I10', 'E11.9' descripcion TEXT NOT NULL, -- Ej: 'Hipertensión esencial (primaria)' grupo VARCHAR(255), -- Ej: 'Enfermedades del sistema circulatorio' search_vector TSVECTOR -- Para optimización de búsqueda (Fuzzy Search) ); -- Indexación para búsqueda rápida por descripción y código CREATE INDEX idx_cie10_search ON CIE10_Catalog USING GIN (search_vector); CREATE INDEX idx_cie10_code_trgm ON CIE10_Catalog USING gist (codigo gist_trgm_ops); ``` ## 2. API Logic (Search Endpoint) **Endpoint:** `GET /api/diagnostics/search?q={query}` - **Logic:** Performs a fuzzy search matching the query against both `codigo` and `descripcion`. - **Optimization:** Limits results to the top 10 most relevant matches. - **Example Response:** ```json [ { "codigo": "I10", "descripcion": "Hipertensión esencial (primaria)", "grupo": "Enfermedades hipertensivas" }, { "codigo": "E11.9", "descripcion": "Diabetes mellitus no insulinodependiente sin complicaciones", "grupo": "Diabetes mellitus" } ] ``` ## 3.
